# 跨设备剪切板同步与临时传输工具

## 项目概述

构建一个 **跨设备剪切板同步 + 临时文件传输工具**，用户无需安装软件，只需通过浏览器即可在不同设备之间快速传输 **文本、图片、文件**。

当前常见的使用方式是：

1. 复制内容
2. 通过聊天软件（如给自己发消息）发送
3. 在另一台设备打开聊天软件
4. 再复制或下载

这种方式存在问题：

- 需要打开额外的软件
- 依赖社交平台
- 操作流程繁琐
- 不适合临时快速传输

本项目的目标是提供一个 **简单、快速、安全的跨设备临时数据通道**。

核心原则：

- 无需安装软件
- 不限制局域网
- 支持文本 / 图片 / 文件
- 数据临时存储并自动过期
- 界面极简，操作快速

---

# 核心模式设计

系统支持 **两种主要模式**。

---

# 模式一：临时传输（无需登录）

该模式用于 **临时向别人或其他设备发送内容**。

## 工作流程

发送方：

1. 打开网页
2. 粘贴文本 / 上传图片 / 上传文件
3. 点击 **发送**
4. 系统生成 **分享码**

接收方：

1. 打开网页
2. 输入 **分享码**
3. 查看 / 复制 / 下载内容

示例流程：

1. 发送方上传代码
2. 系统生成分享码 A7F3K9
3. 接收方输入 A7F3K9 即可查看内容


## 功能特点

- 无需注册或登录
- 临时数据共享
- 默认过期时间：10分钟
- 可选：下载后自动删除

## 支持内容类型

- 文本
- 图片
- 文件

## 分享码规则

随机生成：

- 6～8 位
- 字符范围：A-Z + 0-9

示例：

- A7F3K9
- Q8P1X2

---

# 模式二：设备同步（需要登录）

该模式用于 **个人多设备剪切板同步**。

用户注册账号后，在多个设备登录同一账号，即可实现 **剪切板自动同步**。

## 工作流程

用户在多个设备登录：

- MacBook
- Windows电脑
- 手机
- 平板

当用户在某设备复制内容：

Mac复制内容
↓
服务器接收更新
↓
其他设备获取更新
↓
剪切板自动同步


示例：

- Mac复制代码
- Windows可以直接粘贴


## 同步实现方式

可以采用以下方案之一：

1. WebSocket 实时同步
2. 定时轮询（例如每3秒检查一次）
3. 浏览器插件实现剪切板监听

# 后端架构设计

推荐技术栈：

- 后端：Spring Boot
- 缓存：Redis
- 文件存储：本地磁盘 / 对象存储
- 前端：Vue 或 React
- 部署：Docker + Nginx

---

# Redis 数据设计

临时数据存储在 Redis 中。

Key 结构：

- share:{code}

示例：

- share:A7F3K9

Value 示例：

{
"type": "text",
"content": "hello world"
}

过期时间：

- 600 秒

---

## 文件存储方案

文件不建议直接存 Redis。

推荐方式：

1. 文件存储到本地目录或对象存储

示例：

- /tmp/share-files/


2. Redis 中只存文件元数据

示例：

{
"type": "file",
"filename": "example.zip",
"path": "/tmp/share-files/example.zip"
}

# API 设计

---

## 创建分享

POST /api/share

请求参数：

- type: text | image | file
- content: 内容
- expire: 过期时间（可选）

返回结果：

{
"code": "A7F3K9"
}


---

## 获取分享内容

GET /api/share/{code}


返回示例：

{
"type": "text",
"content": "hello world"
}


如果不存在：

code expired
